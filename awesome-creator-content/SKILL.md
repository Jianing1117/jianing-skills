# 正文设计模块

开头决定有没有人看，正文决定看不看完、看完有没有互动。这个模块做两件事：**结构对不对**，以及**钩子加没加对地方**。

---

## 前置路由

用户贴入脚本后，先问一句：**「你的开头有没有用过 awesome-creator-hook 优化过？」**

- **没有 / 不确定** → 建议先去用 `awesome-creator-hook` 优化开头，完成后再回来做正文设计。开头是前 3 秒的生死线，正文再好也挡不住开头先失去观众。
- **有，或者用户说不用管开头直接继续** → 进入正文设计流程。

---

## 第一步：收集信息

需要两件事：

1. **平台**：发在小红书、抖音，还是两个都发？
2. **脚本内容**：直接粘贴到对话框，或者提供文件路径（Obsidian 文件等）

**为什么先问平台**：五种钩子在不同平台的权重不同，知道平台后才能决定重点加哪几种、比例怎么调。

---

## 第二步：内容结构诊断（Part A）

根据脚本内容，判断属于哪种类型，检查结构是否匹配。

### 五种内容类型和结构原型

| 类型 | 核心结构 | 常见问题 |
|------|---------|---------|
| **观点型** | 结论先行 → 拆解论据 → 案例验证 → 一句话收尾 | 先铺垫太久才说结论，用户等不及 |
| **教程型** | 痛点共鸣 → 分步展开 → 避坑提醒 → 效果预期 | 步骤太密集、没有喘息节奏 |
| **故事型** | 场景代入 → 冲突/困境 → 转折点 → 结果/感悟 | 冲突不够强，转折太平，感悟太说教 |
| **清单型** | 痛点/场景引入 → 逐条展开（每条有小钩子）→ 汇总提炼 | 每条信息密度不均，后半段开始草草收场 |
| **对比型** | 常见做法 → 为什么不行 → 正确做法 → 对比结果 | 「为什么不行」解释太弱，说服力不足 |

**诊断输出**：指出用户的脚本属于哪种类型，当前结构和原型有哪些差距，建议怎么调整。如果结构没问题，直接进入 Part B。

---

## 第三步：钩子设计（Part B）

### 平台权重速查

先告诉用户当前平台下各钩子的优先级，后续插入比例按此调整：

| 钩子类型 | 标注格式 | 小红书权重 | 抖音权重 |
|---------|---------|-----------|---------|
| 完播 | `【完播钩子】` | ★★★ | ★★★★★ |
| 点赞 | `【点赞钩子】` | ★★ | ★★★★ |
| 收藏 | `【收藏钩子】` | ★★★★★ | ★★ |
| 评论 | `【评论钩子】` | ★★★★ | ★★★ |
| 转发 | `【转发钩子】` | ★★ | ★★★ |

- **只发小红书**：重点加收藏钩子和评论钩子，完播钩子按需
- **只发抖音**：重点加完播钩子，点赞钩子每段必有，评论钩子结尾一个
- **两个平台都发**：五种都考虑，收藏和完播各自都要照顾到

---

### 五种钩子说明

---

#### 【完播钩子】— 服务于完播率

核心逻辑：每 15-30 秒给用户一个继续看的理由。放在段落过渡处，不要放在一段话中间。

常见形式：
- 悬念预告：「但还有一个更关键的点，很多人没注意到」
- 数字倒计：「第 3 个方法才是我压箱底的，先说前两个」
- 转折提示：「等一下，这里有个反直觉的东西」
- 时间锚定：「下一分钟是整条视频最重要的部分」

**插入原则**：不要堆砌，节奏要自然，像说话一样带过，不要让人觉得被骚扰。

---

#### 【点赞钩子】— 服务于点赞

核心逻辑：点赞来自情绪共鸣，尤其是「说出了我的心声」「这句话太准了」的瞬间。

常见形式：
- 金句：高度浓缩的一句话，能独立成立，有记忆点
- 情绪共鸣：把用户想说但说不出来的话说出来
- 身份认同：「如果你也是 XXX 类型的人，你一定懂这种感觉」

**放置位置**：开头和结尾必须有；正文里每个核心观点讲完后跟一句。

---

#### 【收藏钩子】— 服务于收藏

核心逻辑：用户收藏的东西是「等会儿/以后用得到」的东西，必须是**可以直接用的结构化内容**。

**重要前置步骤**：在插入收藏钩子之前，先帮用户把内容整理好：
- 如果是清单 → 统一格式，确保每条可扫读（数字编号 + 一句话）
- 如果是步骤 → 明确动词开头，步骤边界清晰
- 如果是框架/方法论 → 考虑对比表格或结构化呈现

收藏钩子形式：
- 截图提示：「建议截图保存这张清单」
- 资源自然提及：「我把这套方法整理成了一份模板，放在主页合集里了」（不要求用户做任何事）
- 价值宣告：「这 5 个步骤我自己用了一年，整理在这里了」

**转发钩子可以跟收藏钩子放在一起**：收藏是为自己留着，转发是送给别人——两个动作可以同时触发。

---

#### 【转发钩子】— 服务于转发

核心逻辑：用户转发，是因为这条内容帮他表达了他想说的话，或者他觉得某个朋友需要看到。

常见形式：
- 帮用户表达：一句话金句，用户看完觉得「这就是我想说的」
- 送给别人：「发给你身边正在 XXX 的朋友」「如果你认识这样的人，转给他」

**放置位置**：结尾，或一个特别有共鸣的金句之后。

---

#### 【评论钩子】— 服务于评论

核心逻辑：用户评论，要么是被问到了，要么是想表达立场，要么是门槛足够低（打一个字就行）。

常见形式（每篇 1-2 个中间 + 1 个结尾）：

| 类型 | 话术模板 | 示例 |
|------|---------|------|
| 开放问题（结尾必须有） | 真实的、和内容相关的提问 | 「你现在最卡在哪一步？评论区说说」 |
| 站队 | 「你是 A 还是 B？」 | 「你觉得你是执行型还是想太多型？」 |
| 选择题 | 「打 1/2/3」 | 「你现在卡在哪？1.不知从哪开始 2.开始了做不下去 3.做完了没人看」 |
| 自测 | 「你有没有过……？」 | 「你有没有发过一条视频，数据好但下一条不知道发啥？有的打个 1」 |
| 挑战 | 「试试看：……」 | 「我挑战你现在就想一下：你上一条发出去的视频，真正帮到了谁？」 |

---

### 合规红线

| 禁止 | 原因 |
|------|------|
| ❌「评论领资料」「扣 1 扣 2」 | 诱导互动，平台违规 |
| ❌「私信我领取」 | 引导私信导流，违规 |
| ❌「点赞收藏评论三连」 | 多重利益交换，严重违规 |
| ✅ 问真实问题 | 自然对话，不涉及利益交换 |
| ✅ 提自己有资源 | 只提存在，不要求用户做任何事 |
| ✅ 挑战/自测 | 用户自愿参与 |

## 输出格式

### 情况 A：用户提供了文件路径（如 Obsidian 文件）

在**同一文件夹**下创建一个新文件，文件名为 `[原文件名]-content-v2.md`，不修改原文件。

文件结构：

```
# [原标题] · 正文优化版

> 平台：[小红书/抖音/双平台]
> 修改说明：
> - 结构调整：[简述做了什么结构改动]
> - 钩子插入：共 X 个（完播 X / 点赞 X / 收藏 X / 转发 X / 评论 X）

---

[正文内容，钩子以标注形式嵌入，例如：]

……这里是第二段内容……

> 【完播钩子】但接下来这个才是关键，先把这个搞清楚

……第三段……

> 【点赞钩子】大多数人在这一步就放弃了，但你坚持到这里了

……
```

### 情况 B：用户直接粘贴到对话框

在对话框里返回一段新文本，格式同上（标注嵌入正文），不需要创建文件。
