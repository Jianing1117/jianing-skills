# 开头优化模块

短视频的前 3 秒是生死线。算法在看完播率，用户在看值不值得继续。开头的唯一任务：**让人不划走**。

---

## 好开头的三个要素

```
好开头 = 话题 + Hook + 可信度
```

- **话题**：我在讲什么。不能假设用户看了标题，开头必须独立工作——第一句话就要让人知道这条视频是关于什么的
- **Hook**：为什么要继续看。数字 / 反差 / 悬念 / 情绪——让用户觉得「等等，这个我要听完」
- **可信度**：为什么要信你。你做到过的成果、亲历的案例、研究过的数据——告诉用户你有资格说这件事

**例子**：
- 话题：选题和标题的区别
- Hook：我去年涨粉 200 万
- 可信度：靠的就是搞清楚这个
- 完整开头：「我去年涨粉 200 万，靠的就是搞清楚选题和标题的区别」

**核心原则**：开头不说答案。把方向点出来，把结论留在后面——有悬念才有人看到最后。

---

## 最大公约数原则

短视频的核心是找到最大公约数。

在生成开头之前，先确认一件事：**这个话题，用最普通的语言来说，能吸引多大的人群？**

话题框架越普世，算法能推到的流量池越大：

- 「做内容创业的人」→ 「想靠兴趣赚钱的人」
- 「学 AI 提效的人」→ 「想少加班的人」
- 「研究理财的人」→ 「不想再为钱焦虑的人」

用大众说得出口的语言，而不是行业黑话。同样的内容，换一个更普世的框架，覆盖面可以扩大 10 倍。

## 工作流程

### 第一步：收集信息

需要两件事：

1. **内容**：视频文案或脚本。内容越完整，生成的开头越有素材、越真实。如果只有标题或选题，也可以先用「悬念制造」方法生成——但脚本写好后建议再回来优化一次，效果更好
2. **现有开头**（如果有）：直接给，一句话点出问题，然后直接进入生成

---

### 第二步：生成开头候选

用三种方法生成开头，总共 **10-15 条**，每种方法 3-5 条。

**口播硬性条件**：所有候选在输出前都必须能流畅说出来。书面语、自问自答（「你有没有想过…其实…」）、绕口长句——直接重写，不输出。

---

## 三种生成方法

---

### 方法一：素材提取

从用户提供的内容里找素材，按优先级顺序使用：

| 素材类型 | 优先级 | 怎么判断 |
|--------|--------|---------|
| 晒结果 + 反转 | ⭐⭐⭐⭐⭐ | 有具体可见的成果，且背后的方法出乎意料 |
| 数字冲击 | ⭐⭐⭐⭐ | 大数字、时间压缩、对比数字 |
| 反差 / 转变 | ⭐⭐⭐⭐ | 之前 vs 之后，差距越大越好 |
| 金句 | ⭐⭐⭐ | 能独立成立的观点，有记忆点 |
| 权威 / 案例 | ⭐⭐⭐ | 名人故事、知名品牌、行业数据 |
| 痛点共鸣 | ⭐⭐⭐ | 目标人群说得出口的真实焦虑 |

基于找到的素材，生成 3-5 条开头。某类素材完全没有就跳过，数量补到有素材的类型上。

**最高优先级示例**（晒结果 + 反转）：
> 「我在过去一个月涨粉了两万多——很多人问我怎么做到的，其实我只是做好了这一件事」
> — 可信度（两万多）+ 悬念（一件事），两件事同时成立

---

### 方法二：素材增补

如果文案里现有素材不够，分两步走：

**第一步：话题普世化检查**

先看当前话题的框架，能不能用更普世的语言重写——这一步有时比补素材更有效，直接换框架就能让开头覆盖更大的人群。

**第二步：向用户提问**

把以下 6 类素材列出来，问用户有没有：

1. **数字**：有没有让人意想不到的数字？（涨了多少 / 花了多少时间 / 多少人犯了这个错）
2. **成果**：你或者你见过的人，有没有做到过某个具体的结果？
3. **转变**：有没有明显的「之前 vs 之后」对比？
4. **金句**：这个内容里，有没有一句话特别有道理、值得记住？
5. **权威**：有没有名人 / 品牌 / 数据来源可以引用？
6. **痛点**：你的目标用户现在最焦虑的一件事是什么，用他们的语言说出来？

用户提供什么就用什么，生成 3-5 条开头。

**如果以上都给不出来**：AI 进入主动补充模式——基于话题方向，主动生成合理的数据或案例。输出时注明「这是补充的素材，建议替换成你自己的真实数据」。

---

### 方法三：悬念制造

直接根据内容主题或标题，生成三种悬念类型的开头。**这一步由 AI 完整写出来，不需要用户填空。**

**悬念类型一：好奇缺口**
提出一个答案不明显的问题，然后不给答案。用户想知道结果，只能继续看。

生成 2-3 条，例如：
- 「善良的李亚鹏，认识半个娱乐圈，30 年为什么赚不到钱？」
- 「一个月存 3000 块，10 年后到底有多少？大多数人算错了」

**悬念类型二：反常识**
先说一个违反直觉的结论，然后停下来。用户想知道「为什么」，只能看下去。

生成 2-3 条，例如：
- 「努力工作，是很多人一直穷下去的核心原因」
- 「一个人赚不到钱，是因为你上班上多了」

**悬念类型三：争议立场**
亮出一个会让人想反驳或站队的观点。不管是反对还是支持，用户都被拉进来了。

生成 1-2 条，例如：
- 「副业这件事，我越来越觉得大多数人根本不应该做」
- 「裸辞，是我这辈子做过的最正确的决定」

## 输出格式

```
## 开头候选

### 方法一：素材提取
**[开头]**
→ [用了什么素材 / 为什么有效]

**[开头]**
→ [说明]

### 方法二：素材增补
**[开头]**
→ [补充了什么素材 / 建议替换成真实数据]

### 方法三：悬念制造
**[好奇缺口开头]** → [悬念机制]
**[反常识开头]** → [反转逻辑]
**[争议立场开头]** → [立场角度]

---

## Top 3 推荐
1. **[开头]** — [推荐理由]
2. **[开头]** — [推荐理由]
3. **[开头]** — [推荐理由]
```

Top 3 选张力最强、口播最自然的 3 条，说清楚推荐理由，选择权交给用户。

---

## 开头自检

选好开头后，逐条过一遍：

1. **「不看标题，我知道这是在讲什么吗？」** — 开头必须独立工作
2. **「前 3 秒有没有让人停下来的理由？」** — 有数字 / 反差 / 悬念 / 情绪其中一项
3. **「我把答案说完了吗？」** — 说完了就没有继续看的理由，重写
4. **「读出来顺不顺口？」** — 把开头大声念一遍，卡壳的地方就是需要改的地方
