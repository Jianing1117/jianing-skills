# 标题优化模块

标题是内容的门面。核心只有一件事：让用户在 0.5 秒内判断"这个值得点开"。

你的工作不是评判用户的标题好不好，而是**给出足够多的候选，让用户自己选**。

---

## 两个平台，两套逻辑

| | 小红书 | 抖音 |
|--|--------|------|
| **首要任务** | 关键词 SEO + 获得感 | 情绪触发 + 好奇缺口 |
| **字数** | ≤20 字（含标点，硬性限制） | 无硬性限制，≤15 字更佳 |
| **关键词** | 前 18 字放 2 个核心词 | 不是 SEO 重点 |
| **检验标准** | 用户看完知道能得到什么 | 用户看完不得不想知道结果 |

---

## 小红书双标题思维

小红书有两个标题，服务于两个完全不同的对象：

**封面标题，给人看。** 它出现在封面图片上，在信息流里承担「让眼神定住、停止滑动」的任务。语言要口语化、情绪化，像钩子一样。用户在还没点进去之前，封面标题已经在工作。

**正文标题，给算法看。** 它是图片下方的文字标题，核心任务是让系统识别内容、精准推给目标用户。关键词要清晰，主题要明确，帮助搜索和推荐机制找到应该看这条内容的人。

| | 封面标题 | 正文标题 |
|--|--------|--------|
| **读者** | 人（刷到的用户） | 算法（搜索 + 推荐系统） |
| **位置** | 封面图片上的大字 | 图片下方的文字标题 |
| **字数** | 5-10 字（越短越有力） | ≤20 字（硬性限制） |
| **关键词** | 不强求，情绪 / 数字 / 反差优先 | 前 18 字放 2 个核心搜索词 |
| **目标** | 让人眼神定住，停止滑动 | 被精准推送给目标用户 |

同一条内容，两个标题可以完全不同——封面写「月入 5 万！」，正文标题写「全职主妇副业变现 90 天实战经验」。一个抓眼，一个抓搜。

**抖音没有这个区分。** 视频封面文字和发布时的标题是同一套逻辑，走情绪触发 + 好奇缺口原则。

---

## 工作流程

### 第一步：收集信息

需要三件事：

1. **内容是什么**：形式不限——可以是一句话描述、一段大纲、一篇完整文章、从 Obsidian 直接粘贴的笔记、或者一段视频脚本。如果用户给了大段内容，自己提取核心话题和关键素材（数据、反差点、结果），不要让用户再重新概括一遍
2. **目标平台**：小红书、抖音，还是两个都要
3. **标题类型**（小红书专属）：需要封面标题（图上的字）、正文标题（图下的文字标题），还是两个都要？如果没有特别说明，默认两个都生成
4. **现有标题**（如果有）：直接给，作为参考基准

如果用户只贴了内容但没说平台，问平台。如果内容本身很清楚，不需要再让用户解释——直接进入生成。

---

### 第二步：生成标题候选

根据内容和平台，覆盖以下 9 种触发模式，**每种模式至少生成 3 个候选**，共 27 个以上。

**小红书·正文标题**：每个候选严格控制在 20 字以内（含标点），前 18 字嵌入 2 个用户真实会搜的关键词。给算法看，精准触达。

**小红书·封面标题**：5-10 字，情绪 / 数字 / 反差优先，不强求关键词，看一眼就懂。给人看，让眼神定住。

**抖音注意**：优先制造情绪张力或好奇缺口，不要把答案说完。

**通用原则**：用普世词（赚钱、涨粉、不上班）替换行业术语（知识付费、引流款、自媒体）。用词越具体普通，能被算法推到的流量池越大。

**张力过滤**：每个候选在输出前自检张力，命中以下至少 2 项才纳入候选，否则重写：
- 有具体数字
- 有对比 / 反差
- 有悬念（标题没把答案说完）
- 有冲突 / 争议
- 有时间承诺
- 有结果承诺

**模式适配**：9 种模式不必强套。如果某个模式跟内容明显不搭（比如日常生活分享很难用恐惧/损失），跳过它，把数量补到更合适的模式上，保证整体质量而不是凑数。

## 九大标题触发模式

---

### 1. 数字结果 + 悬念反差

**核心机制**：用具体数字制造冲击力，再用悬念或反差勾起好奇心。读者看到结果，但不知道怎么做到的——必须点进去。

**公式**：`[起点] + [结果数字]，[悬念/反差]` 或 `[做了一件事]，[数字结果]`

**例**：
- "我如何在 90 分钟赚到 10 万"
- "全职主妇副业 90 天月入 5 万，让你想不到的秘密"
- "犯了这个不同寻常的错，我反而在台上特别自信"

---

### 2. 圈定人群 + 极致结果

**核心机制**：用身份标签、状态、或人生节点精准圈人，同时给出极致结果或消除障碍——相关的人觉得"说的就是我"或"这我也能做到"，不得不点。

**公式**：
- `[人群/状态] + [数字] + [极致结果]`
- `给每个 [状态描述] 的人`
- `如何不 [最大障碍]，就能 [结果]` / `没有 [资源]，也能 [结果]`
- `[人生节点] 之后，你最需要做的 [X]`

**例**：
- "给每个 35+ 职场女性的终极建议"
- "25 岁还没有分文存款的人"
- "不会 IT，也能几分钟搭好你的自建站"

---

### 3. 极致情绪 + 场景共鸣

**核心机制**：用强烈、普遍的情绪词直接命中用户在特定场景下的内心感受。情绪是最快的注意力触发器。

**公式**：`[情绪热词] + [具体场景/结果]`

**例**：
- "月薪 3000 党哭了，这 5 个省钱技巧让我存下 2 万"
- "裁员第三天，我做了这辈子最正确的决定"
- "在北京租房 5 年，终于找到一个不委屈的方法"

---

### 4. 反常识 + 反转叙事

**核心机制**：打破用户已有认知，或用亲身经历制造意料之外的反转——"这不对吧？"和"这怎么可能？"都能让人停下来。适合观点类、科普类、纠错类、个人故事类。

**公式**：`[常识操作] + 其实对你有害 / 没用` 或 `为什么我不 [大多数人都在做的事]` 或 `[负面事件]，反而让我 [正面结果]`

**例**：
- "为什么喝牛奶其实对你一点也不好？"
- "为什么我不玩微信朋友圈"
- "公司破产是如何带给我财富自由和内心平静的"

---

### 5. 好奇缺口

**核心机制**：暗示"有你不知道的信息"，制造信息不对称焦虑，用户点进来是为了消除这个焦虑。

**公式**：`[某群人] 不会告诉你的 [X]` 或 `关于 [话题]，你太晚知道的 [X] 个事` 或 `为什么你 [某行为]，反而 [反效果]？`

**例**：
- "会赚钱的博主不会告诉你的建议"
- "关于互联网营销，创业者们太晚知道的 3 个教训"
- "营销专家们不想要你知道的 5 件事"

---

### 6. 冲突与争议

**核心机制**：亮出一个有争议的立场或事件，加上明确的自我观点，引导用户站队表态。不必是疑问句——强烈的断言同样能激发讨论欲。

**公式**：`[有争议的立场/事件] + [明确观点] + [隐性引导战队]`

**例**：
- "裸辞会不会是你做过的最好决定？"
- "停止学习！！开始实践！！"
- "做微商已经过时了么？"

---

### 7. 指令挑战 + 趣味性

**核心机制**：发出邀请或挑战，激发参与感和行动欲。适合打卡类、挑战类、互动类内容。

**公式**：`敢不敢和我一起 [挑战]？` 或 `跟我一起 [X] 天 [做到某件事]`

**例**：
- "敢不敢测一测，你的脸书主页是不是最优化的"
- "如果你想要赚钱，你需要先对它着迷"
- "至关紧要的头一分钟 - 如何谈判"

---

### 8. 恐惧 / 损失

**核心机制**：激活损失厌恶心理——用户不是冲着"得到"点进来，而是怕"亏了 / 犯错 / 白努力了"。适合避坑类、警示类、纠错类。

**公式**：`[数字] 件阻碍你 [目标] 的事` 或 `[X] 的最根本原因` 或 `警告！这些事正让你的 [努力] 白费`

**例**：
- "3 件阻碍你 IP 变现的事"
- "减肥不成功的最根本原因"
- "警告！这 3 件事正让你的保湿霜涂了等于白涂"

---

### 9. 权威借力

**核心机制**：借用名人 / 案例 / 行业数据提升可信度，让内容"不只是你说的"。适合方法论分享、读书笔记、案例拆解。

**公式**：`[名人/案例] + [话题] + 帮助我 [结果]` 或 `最好的 [一群人] 都做了什么不同的事？` 或 `[一群人] 成功的最根本原因`

**例**：
- "TONY ROBBINS 的晨间冥想帮助我保持心流状态"
- "最赚钱的短线投资人都做了什么不同的事情？"
- "全职妈妈副业成功的最根本原因"

```
## 重点推荐
### 正文标题
1. **[标题]** — [一句话理由]
2. **[标题]** — [一句话理由]
3. **[标题]** — [一句话理由]

### 封面标题（如需）
1. **[封面标题]** — [一句话理由]
2. **[封面标题]** — [一句话理由]
3. **[封面标题]** — [一句话理由]
```

重点推荐中，正文标题从候选里挑张力最强、最符合平台逻辑的 3 个；封面标题挑视觉冲击最强的 3 个。不替用户做最终决定，推荐理由说清楚，选择权交给用户。

## 标题自检（发出去之前问自己）

看完标题，逐条过一遍：

1. **"这和我有什么关系？"** — 有没有点到用户的身份 / 状态 / 痛点
2. **"我讲给谁听？"** — 是否能明确感受到这条内容是为某类人说的
3. **"击中的是真实痛点，还是表象概念？"** — 受众真正想要的底层欲望是什么？"不上班"比"财务自由"更真实，"靠兴趣赚不到钱"比"消费型兴趣"更真实
4. **"他们会不会对这个标题感兴趣？"** — 目标用户看到这个标题，会不会真的想点进去？标题有没有把答案说完？说完了就没有点进去的理由
5. **"话题够普世吗？"** — 用的是大众搜索语言，还是行业黑话？"不上班"比"财务自由"流量池大十倍

五个都能答上来，标题合格。有一个答不上来，换一个候选试试。

## 最后提醒

标题是假设，数据才是验证。同一条内容，多准备几个候选，发出去对比数据——这比死磕一个标题更有价值。
